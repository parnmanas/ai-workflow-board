# Quick Task 260412-2r8: Fix agent-online flapping — guard markOffline Summary

Single-method edit guarding `AgentConnectionService.markOffline` with a `last_seen_at` freshness check in the SQL WHERE clause, so Claude CLI's per-request Streamable HTTP sessions (DELETE /mcp immediately after each ping) can no longer overwrite a fresh `is_online=1` written milliseconds earlier by the ping tool.

## Change Overview

**File:** `apps/server/src/modules/agents/agent-connection.service.ts`

`markOffline(agentId)` previously issued an unconditional `UPDATE agent SET is_online=0 WHERE id=:id`. Because Claude CLI opens a new MCP session per JSON-RPC batch and issues `DELETE /mcp` right after each response, `transport.onclose` fired constantly and stamped agents OFFLINE milliseconds after the same session's `ping` tool had stamped them ONLINE. The dashboard therefore showed agents flapping even though they were heartbeating every 30s.

The fix computes `threshold = now - OFFLINE_THRESHOLD_MS` (60s, reused from the existing sweep constant) and extends the WHERE clause to:

```
id = :id AND (last_seen_at IS NULL OR last_seen_at < :threshold)
```

This means:
- A recent ping (fresh `last_seen_at` within 60s) is never overwritten — the UPDATE matches 0 rows and the method becomes a silent no-op.
- A genuinely stale session closing still marks the agent offline immediately (the `last_seen_at < :threshold` branch).
- A freshly-created agent that never pinged still gets marked offline on `transport.onclose` (the `last_seen_at IS NULL` branch) — behavior parity with the old method for that edge case.
- The `sweepOfflineAgents` 30s interval remains the canonical offline detector for agents that silently drop without `transport.onclose` ever firing.

The check lives inside a single SQL expression (not a read-then-branch JS check) so it is atomic with respect to a concurrent `ping` save on the same row — there is no TOCTOU window for the race to slip through.

JSDoc was updated to explain the Claude CLI session-per-request behavior and point at `sweepOfflineAgents` as the canonical offline detector.
