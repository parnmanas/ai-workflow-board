<task type="auto">
  <name>Task 1: Fix .awb-shell and .awb-content CSS in main.tsx</name>
  <files>apps/client/src/main.tsx</files>
  <action>
In apps/client/src/main.tsx, modify the injected CSS for the `.awb-shell` class:

1. Change `min-height: 100vh` to `height: 100vh` — this locks the shell to exactly the viewport height instead of allowing it to grow with content.

2. Add `overflow: hidden` to `.awb-shell` — this prevents any scroll on the outermost container. The sidebar and header are children of this container and will stay fixed.

3. Change `.awb-content` from `overflow: hidden` to `overflow-y: auto` — this is the main content area (renders `<Outlet />`) and must be the ONLY element that scrolls vertically. The Board page, Admin page, etc. all render inside this container.

The resulting CSS should be:
```css
.awb-shell {
  display: flex;
  height: 100vh;
  overflow: hidden;
  background: ${tokens.colors.surface};
}
```
```css
.awb-content {
  flex: 1;
  overflow-y: auto;
}
```

Leave `.awb-main` unchanged — it already has `overflow: hidden` and `flex: 1` which is correct (it prevents the main column from expanding beyond the shell).

In ChatPage.tsx, the outer containers for both mobile and desktop layouts already use `height: '100%'` and `display: 'flex'` / `flexDirection: 'column'`. However, the outer div needs `overflow: hidden` to prevent ChatPage content from bubbling scroll up to `.awb-content`.

1. **Desktop layout** (line ~2345): The outer `<div>` wrapping the `<Group>` already has `height: '100%'` and `flexDirection: 'column'`. Add `overflow: 'hidden'` to this div's style:
   ```
   style={{ height: '100%', display: 'flex', flexDirection: 'column', overflow: 'hidden' }}
   ```

2. **Mobile layout** (line ~2301): Same fix — add `overflow: 'hidden'` to the mobile outer div:
   ```
   style={{ height: '100%', display: 'flex', flexDirection: 'column', overflow: 'hidden' }}
   ```

3. **Group component** (line ~2347): The `<Group>` already has `style={{ height: '100%' }}`. Verify this is present. The Group component from react-resizable-panels should expand to fill the remaining flex space. Add `flex: 1` and `overflow: 'hidden'` to ensure it doesn't exceed its container:
   ```
   style={{ height: '100%', flex: 1, overflow: 'hidden' }}
   ```

The ChatRoomListPanel (line ~686) and ChatRoomView (line ~1598) already have `height: '100%'` + `overflow: 'hidden'` on their root divs, and their scrollable children already use `overflowY: 'auto'`. These are correct and need no changes.
